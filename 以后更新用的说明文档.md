# 日历音乐双卡片项目更新说明

## 一、当前文件结构

本项目使用 3 个独立 HTML 文件：

1. `整合1.2版日历音乐播放器_点击上传_外部文件版.html`
   - 主入口文件
   - 负责日历/音乐双卡片切换
   - 引用另外两个文件

2. `calendar.html`
   - 日历卡片内容
   - 日历样式、日期显示、卡片交互逻辑都在这里

3. `music_player_new.html`
   - 音乐播放器内容
   - 包含点击封面上传、清空图片、自定义封面轮播等功能

---

## 二、以后更新时改哪个文件

### 1. 只改音乐播放器功能
修改 `music_player_new.html`

适用场景：
- 调整封面上传
- 修改清空图片按钮
- 调整轮播逻辑
- 改播放器样式
- 改播放控制相关功能

### 2. 只改日历功能
修改 `calendar.html`

适用场景：
- 调整日历样式
- 修改日期显示
- 改卡片布局
- 改日历动画
- 改日程/公告内容

### 3. 只改外层主页面
修改 `整合1.2版日历音乐播放器_点击上传_外部文件版.html`

适用场景：
- 修改日历/音乐切换逻辑
- 改双卡片容器布局
- 改页面标题
- 改 iframe 引用路径
- 改外层动画效果

---

## 三、更新流程

### 方法1：直接替换单个文件
如果只改一个功能，只需要把对应文件重新上传到 GitHub 仓库即可。

例如：
- 改播放器 → 替换 `music_player_new.html`
- 改日历 → 替换 `calendar.html`
- 改外层页面 → 替换主入口文件

### 方法2：本地修改后重新上传
1. 让助手生成新版本文件
2. 下载或复制到本地
3. 覆盖上传到 GitHub
4. 等 GitHub Pages 自动刷新

---

## 四、文件名建议

### 建议保持文件名不变
尽量继续使用这三个文件名：
- `calendar.html`
- `music_player_new.html`
- `整合1.2版日历音乐播放器_点击上传_外部文件版.html`

这样主文件里的引用不用改，最稳定。

### 如果文件名改了
要同步修改主文件中的引用路径，例如：
```html
<iframe src="music_player_new.html"></iframe>
```

如果改名，就必须同时改这里。

---

## 五、GitHub Pages 更新注意事项

### 1. 缓存问题
GitHub Pages 有时不会立刻更新，可能会看到旧页面。

解决方法：
- 强制刷新页面
- 等 1～3 分钟
- 重新打开链接

### 2. 路径要一致
三个文件必须放在**同一个目录**下，否则 iframe 可能找不到文件。

### 3. 文件编码
建议统一使用 UTF-8 编码，避免中文文件名和中文内容乱码。

---

## 六、推荐的维护习惯

以后每次改功能时，最好先确认：

1. 改的是哪个文件
2. 是否需要同步改主页面引用
3. 上传后是否要等缓存刷新
4. 页面是否能正常打开
